## Research Objective 3: Error Taxonomy

### Error Type 1: Domain Confusion (Wrong L1)
- **Severity**: High
- **Frequency**: ~1.5% of total classifications
- **Source stages**: Stage 2 (HHS → wrong subdomain), Stage 3a (cross-domain keywords)
- **Example**: HHS OCR guidance → `clinical-operations` instead of `hipaa-privacy`
- **Detection**: 
  ```sql
  SELECT COUNT(*) FROM kg_classification_log 
  WHERE agency_slug LIKE 'hhs-%' 
    AND domain_code NOT LIKE 'hipaa-%' 
    AND confidence > 0.7
  ```
- **Mitigation**: Post-classification validation rule: HHS OCR → must be HIPAA domain

### Error Type 2: Subdomain Confusion (Right L1, Wrong L2/L3)
- **Severity**: Medium
- **Frequency**: ~4% of total classifications
- **Source stages**: Stage 3a (sibling keyword overlap), Stage 4 (fuzzy boundaries)
- **Example**: FDA warning letter → `fda-regulation.drugs` instead of `fda-regulation.enforcement`
- **Detection**: Track correction patterns where L1 unchanged but L2 changed
- **Mitigation**: 
  - Negative keywords: `drug` + `warning letter` → enforcement not drugs
  - Stage 4 prompt: explicit enforcement vs product category guidance

### Error Type 3: Missing Cross-Classification
- **Severity**: Medium
- **Frequency**: ~15% of applicable entities (per S7-C patterns)
- **Source stages**: Stage 3a limited keyword coverage for secondary domains
- **Example**: Telemedicine DEA guidance → has `controlled-substances` but missing `telehealth`
- **Detection**: 
  ```sql
  WITH trigger_patterns AS (
    SELECT source_domain, target_domain, trigger_keywords 
    FROM s7c_triggers
  )
  SELECT e.entity_id
  FROM kg_entities e
  JOIN kg_entity_domains ed ON e.id = ed.entity_id
  JOIN trigger_patterns t ON ed.domain_code = t.source_domain
  WHERE NOT EXISTS (
    SELECT 1 FROM kg_entity_domains ed2 
    WHERE ed2.entity_id = e.id 
      AND ed2.domain_code = t.target_domain
  )
  ```
- **Mitigation**: S7-C trigger enforcement in Stage 3a; Stage 4 few-shot examples

### Error Type 4: Over-Classification
- **Severity**: Low-medium
- **Frequency**: ~3% of multi-domain entities
- **Source stages**: Stage 3a greedy matching, Stage 4 hedging
- **Example**: General FDA announcement → 5 domains when only 2 relevant
- **Detection**: Flag entities with 4+ domains where secondary confidence < 0.6
- **Mitigation**: 
  - P2-S2's max 4 secondary domains cap
  - Confidence threshold for secondary assignment: 0.50 minimum

### Error Type 5: Confidence Miscalibration
- **Severity**: High (bypasses review)
- **Frequency**: ~1% of high-confidence classifications
- **Source stages**: Stage 1 (1.0 for ambiguous parts), Stage 4 (overconfident on familiar patterns)
- **Example**: 21 CFR 866 entity at 1.0 confidence despite S8-A refinement flag
- **Detection**: QA sampling of confidence > 0.90 classifications
- **Mitigation**: 
  - Cap Stage 1 confidence at 0.95 for S8-A's 12 parts
  - Stage 4 temperature = 0.3 (not 0) to maintain uncertainty

## Research Objective 4: Human Review Priority

### 4A. Priority Scoring

```
priority = 0.4 × (1 - confidence) + 0.3 × impact + 0.2 × confusion_risk + 0.1 × recency
```

Where:
- `confidence`: AI classification confidence [0-1], inverted so low confidence = high priority
- `impact`: normalized practice impact score = affected_practices / 14
- `confusion_risk`: 1.0 if in known confusion pair OR model uncertainty_flag=true, else 0
- `recency`: (30 - days_since_created) / 30, capped at [0,1]

**Normalization**: All components scaled to [0,1], final score in [0,1] where 1 = highest priority

### 4B. Review Effort Estimates

Based on weighted accuracy calculations:

| Target Accuracy | Errors to Correct | Reviews Needed* | Hours (2 min/entity) |
|---|---|---|---|
| 90% overall | 9,900 → 4,950 | 1,238 | 41 hours |
| 95% overall | 9,900 → 950 | 4,950 | 165 hours |
| 98% overall | 9,900 → 198 | 9,702 | 323 hours |

*Assumes 80% review accuracy (reviewer correctly identifies and fixes errors)

Initial corpus only. Monthly maintenance much lower:
- 95% accuracy: ~75 reviews/month = 2.5 hours
- 98% accuracy: ~300 reviews/month = 10 hours

### 4C. Diminishing Returns

**Review ROI Analysis**:
- **First 500 reviews**: Fix systematic errors (agency mappings, keyword gaps). Each correction potentially fixes 10-50 similar entities. **High ROI**
- **500-2,000 reviews**: Fix edge cases and refine boundaries. Each correction helps 2-5 similar entities. **Moderate ROI**
- **2,000-5,000 reviews**: Mostly confirming correct classifications, catching rare errors. **Low ROI**
- **5,000+ reviews**: Negligible improvement, mostly confirming already-correct high-confidence classifications. **Negative ROI** (opportunity cost)

**Recommendation**: Target 1,500 reviews of initial corpus (50 hours over 2 weeks), then 100 reviews/month ongoing. This achieves ~96% accuracy with reasonable effort.

**Dashboard queries**:
```sql
CREATE VIEW classification_metrics AS
SELECT 
  DATE(created_at) as date,
  stage,
  COUNT(*) as volume,
  AVG(confidence) as avg_confidence,
  SUM(api_cost_usd) as total_cost,
  COUNT(DISTINCT entity_id) as unique_entities,
  COUNT(*) FILTER (WHERE needs_review) as flagged_for_review,
  AVG(EXTRACT(EPOCH FROM (completed_at - started_at))) as avg_latency_seconds
FROM kg_classification_log
WHERE created_at > NOW() - INTERVAL '30 days'
GROUP BY 1, 2;

CREATE MATERIALIZED VIEW confusion_matrix AS
SELECT 
  original_domain,
  corrected_domain,
  COUNT(*) as correction_count,
  MAX(created_at) as last_seen,
  ARRAY_AGG(DISTINCT entity_id) as example_entities
FROM classification_corrections
WHERE created_at > NOW() - INTERVAL '90 days'
GROUP BY 1, 2
HAVING COUNT(*) >= 3;
```

## Summary

This cost model and integration reference demonstrates that P2's classification framework is highly cost-effective:

- **Initial corpus**: $37 total API cost (compared to $111 with only 22 rules)
- **Ongoing costs**: $7-14/month for typical daily volumes
- **Accuracy**: 94.4% before review, 96%+ after moderate HITL effort
- **Implementation**: 8 PRPs with clear dependencies and integration points
- **State-ready**: Architecture accommodates jurisdiction-specific rules without modification

The system achieves S3's vision of fast, accurate classification while maintaining full explainability and keeping AI costs minimal through effective rule-based filtering.
