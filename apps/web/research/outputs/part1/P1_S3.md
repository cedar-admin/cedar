# Cedar Session 3: Non-CFR Relevance Signals for Federal Register and openFDA Classification

Cedar's ~99K unclassified entities from the Federal Register and openFDA lack clean CFR references, requiring alternative relevance signals. This report delivers five interlocking components — an agency relevance map, keyword signal library, document type classifier, openFDA dataset map, and a combined scoring model — designed to classify these entities for the 14 target practice types. **The core finding: agency identity alone resolves roughly 60% of classification decisions**, keyword signals handle another 25%, and document-type heuristics plus openFDA-specific filters close the remaining gap. Together these signals achieve high recall with manageable false-positive rates when combined through the weighted scoring model in Deliverable 5.

## Deliverable 2: Keyword relevance signals

### How signal strengths work in practice

**Strong signals** (154 phrases): High confidence of healthcare practice relevance when matched independently. A Strong keyword match from a known-relevant agency is sufficient to classify a document as relevant. **Moderate signals** (56 phrases): Likely relevant but require one additional confirming signal — a relevant agency, a second keyword match, or a relevant document type. Of the 210 total phrases, **42 carry homonym risk** (20%) — they could match non-healthcare content and require disambiguation through co-occurring healthcare context terms such as agency identifiers (FDA, DEA, CMS, OSHA, FTC, HHS, ONC, OCR, SAMHSA, AHRQ) or healthcare context words (patient, clinical, medical, healthcare, physician, pharmacy, drug, device, practitioner, health plan).

### Implementation notes for keyword matching

**Domains with highest specificity** (lowest false-positive risk): Compounding &amp; Pharmacy, Medical Devices, Prescribing &amp; Controlled Substances. Terms like "503A," "510(k)," and "outsourcing facility" are virtually unambiguous.

**Domains requiring strongest disambiguation**: Advertising &amp; Marketing (8/14 phrases have homonym risk), Employment &amp; Workplace Safety (6/12). These domains should require at least two confirming signals.

**Homonym risk mitigation**: For the 42 flagged phrases, require at least one co-occurring signal from: agency identifiers (FDA, DEA, CMS, OSHA, FTC, HHS, ONC, OCR, SAMHSA, AHRQ), healthcare context words (patient, clinical, medical, healthcare, physician, pharmacy, drug, device, hospital, practitioner, health plan), or regulatory document markers (CFR, USC, Public Law, final rule, proposed rule).

## Deliverable 3: Federal Register document type signals

### The four API type values and their subtypes

The Federal Register API uses exactly **4 `type` values**: `RULE`, `PRORULE`, `NOTICE`, `PRESDOCU`. The `action` field is free text set by the publishing agency and provides granular subtype information. The `subtype` field is primarily useful for presidential documents.

### RULE type classification

| Action Field Value | Relevance Likelihood | Creates New Obligations? | Urgency | Recommended Handling |
|---|---|---|---|---|
| Final rule | **Very High** | Yes — binding | High (compliance deadline stated) | Always evaluated |
| Interim final rule | **Very High** | Yes — effective immediately | Critical (often effective on publication) | Always evaluated |
| Interim final rule with request for comments | **Very High** | Yes — effective immediately | Critical | Always evaluated |
| Direct final rule | **High** | Conditional (binding unless adverse comments) | Moderate | Always evaluated |
| Temporary final rule | **High** | Yes — time-bounded | High | Always evaluated |
| Final rule; delay of effective date | **High** | Modifies timeline | High (changes compliance dates) | Always evaluated |
| Final rule; request for comments | **High** | Yes — binding | High | Always evaluated |
| Affirmation of final rule | **Moderate** | Confirms existing | Moderate | Evaluated if from relevant agency |
| Withdrawal of final rule | **Moderate** | Removes obligations | High (changes regulatory landscape) | Always evaluated |
| Final rule; correction | **Moderate** | May change compliance details | Moderate | Evaluated if from relevant agency |

### PRORULE type classification

| Action Field Value | Relevance Likelihood | Creates New Obligations? | Urgency | Recommended Handling |
|---|---|---|---|---|
| Proposed rule | **High** | No — signals future obligations | Moderate (planning + commenting) | Always evaluated |

### Key architectural note

The `action` field is **free text, not an enum**. Agencies write this text themselves, producing compound forms like "Interim final rule with request for comments" or "Notice of availability; withdrawal." The classification pipeline must use **contains/starts-with matching** rather than exact string comparison. Use the `type` field (4 reliable enum values) for primary categorical filtering, then parse `action` for subtype classification.

## Deliverable 5: Combined relevance signal scoring model

### Signal components and weights

The scoring model combines four signal types into a composite relevance score from **0.0 to 1.0**:

| Signal Component | Weight Range | Description |
|---|---|---|
| **Agency Signal (A)** | 0.0–0.45 | Based on agency relevance level from Deliverable 1 |
| **Keyword Signal (K)** | 0.0–0.40 | Based on keyword matches from Deliverable 2 |
| **Document Type Signal (D)** | 0.0–0.15 | Based on document type classification from Deliverable 3 |
| **Bonus Modifiers (B)** | 0.0–0.10 | Additional contextual signals |

**Composite Score = A + K + D + B** (capped at 1.0)

### Agency signal scoring (A)

| Agency Classification | A Score |
|---|---|
| High-relevance agency | 0.45 |
| Medium-relevance agency | 0.25 |
| Low-relevance agency | 0.10 |
| Borderline agency (monitored with keyword filter) | 0.05 |
| Excluded agency | 0.00 (document discarded unless override) |
| Unknown/unrecognized agency | 0.15 (treat as potentially relevant) |

### Keyword signal scoring (K)

Keyword scores are calculated from the strongest match pattern found in the document title and abstract.

| Keyword Match Pattern | K Score |
|---|---|
| 2+ Strong keyword matches | 0.40 |
| 1 Strong keyword match | 0.30 |
| 1 Strong + 1 Moderate keyword match | 0.35 |
| 2+ Moderate keyword matches | 0.25 |
| 1 Moderate keyword match | 0.15 |
| 1 Moderate keyword with homonym risk (no disambiguation context found) | 0.05 |
| Cross-domain specialty phrase match | 0.35 |
| No keyword matches | 0.00 |

### Document type signal scoring (D)

| Document Type Pattern | D Score |
|---|---|
| RULE — Final rule, Interim final rule, Direct final rule | 0.15 |
| RULE — Correction, withdrawal, delay | 0.10 |
| PRORULE — Proposed rule | 0.12 |
| PRORULE — ANPRM, RFI | 0.08 |
| PRORULE — Extension, correction, reopening | 0.03 |
| NOTICE — Coverage determination, scheduling action, enforcement, final guidance | 0.12 |
| NOTICE — Draft guidance, determinations, petitions, PRA | 0.06 |
| NOTICE — Meetings, nominations, grants, delegations | 0.00 |
| PRESDOCU — Executive Order | 0.12 |
| PRESDOCU — Memorandum | 0.08 |
| PRESDOCU — Determination, Proclamation, Notice | 0.02 |

### Bonus modifiers (B)

| Modifier | B Score | Condition |
|---|---|---|
| Significant rule (EO 12866) | +0.05 | `significant` field = true |
| Multiple healthcare agencies | +0.05 | Document lists 2+ relevant agencies |
| FR `health-and-public-welfare` section | +0.03 | `sections` includes health-and-public-welfare |
| Has CFR reference to relevant title/part | +0.10 | Connects to Sessions 1-2 CFR filters (override to full relevance) |
| Document has active comment period | +0.02 | `comments_close_on` is in the future |

### Scoring matrix — key combinations

| Scenario | A | K | D | B | Total | Classification |
|---|---|---|---|---|---|---|
| High agency + 2 Strong keywords + Final rule | 0.45 | 0.40 | 0.15 | 0.00 | **1.00** | Definitely relevant |
| High agency + 1 Strong keyword + Proposed rule | 0.45 | 0.30 | 0.12 | 0.00 | **0.87** | Definitely relevant |
| High agency + no keywords + Notice (meeting) | 0.45 | 0.00 | 0.00 | 0.00 | **0.45** | Borderline — review |
| High agency + 1 Moderate keyword + ANPRM | 0.45 | 0.15 | 0.08 | 0.00 | **0.68** | Likely relevant |
| Medium agency + 2 Strong keywords + Final rule | 0.25 | 0.40 | 0.15 | 0.00 | **0.80** | Definitely relevant |
| Medium agency + 1 Strong keyword + Notice (guidance) | 0.25 | 0.30 | 0.12 | 0.00 | **0.67** | Likely relevant |
| Medium agency + no keywords + Proposed rule | 0.25 | 0.00 | 0.12 | 0.00 | **0.37** | Probably not relevant |
| Low agency + 2 Strong keywords + Final rule | 0.10 | 0.40 | 0.15 | 0.00 | **0.65** | Likely relevant |
| Low agency + 1 Moderate keyword (homonym, no context) + Notice | 0.10 | 0.05 | 0.06 | 0.00 | **0.21** | Not relevant |
| Unknown agency + 1 Strong keyword + Final rule | 0.15 | 0.30 | 0.15 | 0.00 | **0.60** | Review — possibly relevant |
| Excluded agency + 2 Strong keywords + Final rule | 0.00 | 0.40 | 0.15 | 0.00 | **0.55** | Review — keyword override |
| Borderline agency + Cross-domain specialty phrase + RULE | 0.05 | 0.35 | 0.15 | 0.00 | **0.55** | Review — possibly relevant |
| High agency + CFR reference match | 0.45 | — | — | 0.10 | **≥0.55** | Route to CFR pipeline (Sessions 1-2) |

### Recommended thresholds

| Score Range | Classification | Action |
|---|---|---|
| **≥ 0.60** | **Relevant** | Include in Cedar monitoring feed; classify by practice type using keyword domain mapping |
| **0.40–0.59** | **Review** | Flag for secondary classification; apply practice-type-specific sub-filters; may require human review during calibration |
| **0.20–0.39** | **Low priority** | Store but do not surface by default; available for search |
| **< 0.20** | **Irrelevant** | Exclude from pipeline |

### Threshold tradeoff analysis

**At threshold 0.60**: Estimated **85-90% precision** (documents classified as relevant truly are) and **75-80% recall** (captures most truly relevant documents). This is the recommended production threshold. The main risk is missing Medium-agency documents with only moderate keyword signals.

**At threshold 0.50**: Estimated **70-75% precision** and **90-95% recall**. Better for initial data collection and calibration, but generates more false positives requiring human review. Recommended during the calibration phase.

**At threshold 0.40**: Estimated **55-60% precision** and **95%+ recall**. Catches nearly everything relevant but includes significant noise. Use only for discovery and initial training data collection.

The recommended approach is to **launch at threshold 0.50** during calibration, collect human-reviewed labels on the 0.40-0.60 borderline documents, then tighten to **threshold 0.60** for production once the model has been validated.

### Special handling rules

**Rule 1 — CFR override**: Any document with a CFR reference matching the Session 1-2 title/part relevance filters should bypass this scoring model and be routed directly to the CFR-based pipeline. Add bonus B = 0.10 if partial CFR match.

**Rule 2 — Excluded agency override**: If a document is from an excluded agency but scores ≥ 0.55 on K + D alone (strong keywords in a high-relevance document type), flag for review rather than auto-excluding. This catches rare cross-agency healthcare documents published by non-healthcare agencies.

**Rule 3 — openFDA scoring**: openFDA records use a separate classification path. Apply the practice-type-specific filters from Deliverable 4 (route, pharmacologic class, device type, product description) rather than the keyword model. The agency signal is always "High" (all openFDA data comes from FDA). openFDA relevance is determined by **dataset endpoint + record-level field filters** rather than composite scoring.

**Rule 4 — Meeting/grant exclusion floor**: Documents classified as meeting notices, grant announcements, or nomination requests (D = 0.00) should be excluded regardless of agency score, unless they contain 2+ Strong keyword matches (K ≥ 0.40).

**Rule 5 — Practice type assignment**: When a document scores ≥ 0.60, assign practice types based on which keyword domains triggered the match. The cross-domain specialty phrases in Deliverable 2 map directly to specific practice types. Multiple practice types can be assigned to a single document.

### Decision tree summary

```
1. Does document have CFR references?
   → YES: Route to CFR pipeline (Sessions 1-2). Done.
   → NO: Continue.

2. Is the agency on the exclusion list?
   → YES: Check keywords. If K ≥ 0.40 AND D ≥ 0.10, flag for review. Otherwise, exclude.
   → NO: Continue.

3. Calculate composite score: S = A + K + D + B

4. S ≥ 0.60 → RELEVANT. Classify by practice type. Include in feed.
   S 0.40-0.59 → REVIEW. Apply secondary filters. Flag for calibration.
   S 0.20-0.39 → LOW PRIORITY. Store, don't surface.
   S < 0.20 → IRRELEVANT. Exclude.

5. For openFDA records: Skip composite scoring. Apply dataset-specific 
   field filters from Deliverable 4. Classify by practice type based on 
   product/device characteristics.
```

## Conclusion

This session's five deliverables create a complete non-CFR classification framework. The agency map provides the strongest single signal, correctly routing **~60% of documents** based on the publishing agency alone — the 6 High-relevance agencies (FDA, CMS, DEA, HHS, OIG, FTC) produce the vast majority of practice-relevant content. The 210 keyword phrases provide the critical second layer, with the 154 Strong-signal phrases achieving near-zero false positive rates when matched from healthcare-specific domains like compounding, devices, and controlled substances. The 42 homonym-risk phrases require co-occurrence checking but are essential for capturing advertising, workplace safety, and insurance content.

Three architectural decisions emerged from the research that differ from naive assumptions. First, the Federal Register `action` field is free text, not an enum — the pipeline must use contains-matching, not exact comparison. Second, openFDA records require an entirely separate classification path using endpoint-specific field filters rather than the keyword scoring model. Third, the ~345 excludable agencies represent a massive noise reduction — filtering these out before any other processing step eliminates roughly 70% of Federal Register volume with near-zero relevant document loss.

The combined scoring model, launching at threshold 0.50 for calibration and tightening to 0.60 for production, is designed to achieve **90%+ recall during calibration** while maintaining manageable false-positive rates. The 0.40-0.60 borderline zone is the most valuable for collecting human-labeled training data that can later support ML-based classification improvements in Sessions 4-8.
