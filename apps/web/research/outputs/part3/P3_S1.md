# Florida Administrative Code → Cedar Classification Framework: Complete Mapping

**The Florida Administrative Code encodes agency identity directly in its citation structure, giving Cedar a powerful structural advantage for classification.** Every FAC rule under Title 64 (DOH) and Title 59 (AHCA) carries embedded metadata — rulemaking authority, law implemented, effective date, and amendment history — that provides free, high-confidence classification signals. This document maps all nine targeted boards/agencies to Cedar's domain taxonomy across 150+ chapter-level structural units, defines FL-specific keywords, cross-classification patterns, federal reference detection logic, and a complete citation parser specification.

---

## Section 1: FAC chapter to Cedar domain mapping

### 1.1 Verified FAC structure and metadata

The FAC citation format is **Title + Division + Chapter + Rule**, encoded as `64B8-9.003`. The "B" in `64B8` denotes the Division of Medical Quality Assurance within the Department of Health; the trailing numeral `8` identifies the Board of Medicine specifically. Every rule page on flrules.org exposes five structured metadata fields that serve as free classification signals:

| Metadata Field | Description | Classification Value |
|---|---|---|
| **Rulemaking Authority** | Florida Statutes granting rulemaking power (e.g., "458.309 FS") | Maps board to governing statute → confirms domain |
| **Law Implemented** | Florida Statutes the rule implements (e.g., "456.061, 458.331(1) FS") | Identifies specific regulatory intent → refines subdomain |
| **Effective Date** | Date current version took effect | Recency signal for priority scoring |
| **History** | Full chronological amendment trail with former rule numbers | Tracks renumbering (21M → 61F6 → 59R → 64B8) |
| **References** | Incorporated materials/federal regulations | Detects federal cross-references for cross-classification |

These metadata fields appear consistently on every rule detail page at `https://flrules.org/gateway/ruleno.asp?id=[rule]`. The rulemaking authority and law implemented fields are hyperlinked to Florida Statutes, enabling automated extraction.

### 5.2 Inheritance vs. boost decision logic

When an FL rule references a federal regulation, the system should apply the following decision tree:

**Rule: Federal references should BOOST confidence in an already-suggested domain, not independently create cross-classifications.**

The rationale: FL rules reference federal regulations to define their own state-level requirements. The FL rule's own content should drive primary classification; the federal reference confirms the domain is correct.

| Scenario | Action | Confidence Adjustment |
|---|---|---|
| FL rule already classified to domain X; federal reference confirms domain X | **Boost** primary domain confidence | +0.10 to domain X score |
| FL rule classified to domain X; federal reference points to domain Y | **Add cross-classification** to domain Y | Domain Y gets score = 0.15 (low-confidence cross-class) |
| FL rule has no strong primary signal; federal reference is only signal | **Inherit** federal domain as primary | Domain from federal reference gets score = 0.25 (moderate) |
| FL rule incorporates federal regulation by reference (formal incorporation) | **Strong cross-classification** | Federal domain gets score = 0.30 (this is a formal legal adoption) |

## Section 6: FL citation parser specification

### 6.1 FAC citation format definition

The Florida Administrative Code citation format, governed by **FAC Rule 1-1.008**, encodes the complete organizational hierarchy:

```
[Title][DivisionLetter][DivisionNumber]-[Chapter].[Rule]
```

**Formal citation forms:**
- Full: `Rule 64B8-9.003, Florida Administrative Code`
- Abbreviated: `Rule 64B8-9.003, F.A.C.`
- Bluebook: `Fla. Admin. Code R. 64B8-9.003`

### 6.2 Component parsing specification

| Component | Position | Format | Description | Examples |
|---|---|---|---|---|
| **title** | Prefix (1-2 digits) | `\d{1,2}` | Agency/department number | `64` (DOH), `59` (AHCA), `1` (Dept of State) |
| **division_letter** | After title (0-2 uppercase letters) | `[A-Z]{0,2}` | Major activity/division within agency | `B` (MQA boards), `A` (AHCA licensing), `GG` (Psychotherapy) |
| **division_number** | After division letter (0-2 digits) | `\d{0,2}` | Specific board/sub-unit within division | `8` (Board of Medicine), `16` (Board of Pharmacy), `` (none for 59A) |
| **separator** | Between division and chapter | `-` | Hyphen | `-` |
| **chapter** | After hyphen (1-3 digits) | `\d{1,3}` | Functional topic area | `9` (Standards of Practice), `27` (Pharmacy Practice) |
| **decimal** | Between chapter and rule | `.` | Period | `.` |
| **rule** | After decimal (3-5 digits) | `\d{3,5}` | Individual rule identification | `003`, `0091`, `10801` |

### 6.3 Parsing regex

**Primary regex for FAC citations:**
```regex
(?P<title>\d{1,2})(?P<div_letter>[A-Z]{0,2})(?P<div_number>\d{0,2})-(?P<chapter>\d{1,3})\.(?P<rule>\d{3,5})
```

**Contextual regex (matches formal citation in text):**
```regex
(?:Rule\s+|Fla\.\s+Admin\.\s+Code\s+R?\.\s*)?(\d{1,2}[A-Z]{0,2}\d{0,2}-\d{1,3}\.\d{3,5})(?:\s*,?\s*(?:F\.A\.C\.|Florida\s+Administrative\s+Code))?
```

**Subsection reference regex (for internal cross-references):**
```regex
(?:subsection|paragraph|subparagraph)\s+(\d{1,2}[A-Z]{0,2}\d{0,2}-\d{1,3}\.\d{3,5})\((\d+)\)(?:\(([a-z])\))?
```

### 6.4 Edge cases and normalization rules

| Edge Case | Example | Normalization Rule |
|---|---|---|
| No division letter | `59A-4.110` → division is just `A`, no division_number | If `div_letter` is present but `div_number` is empty, the full division identifier is `title + div_letter` (e.g., `59A`) |
| Double division letter | `59GG-1.xxx` | Allow 1-2 uppercase letters in `div_letter` field |
| 4-5 digit rule numbers | `64B16-28.10801` | Rule numbers can be 3-5 digits; leading zeros are significant (`.003` ≠ `.3`) |
| Emergency rules | `64B8ER22-3` | Separate pattern: `(\d{1,2}[A-Z]{0,2}\d{0,2})ER(\d{2})-(\d{1,3})` — extract board + year + chapter |
| Historical rule numbers | `21M-27.003`, `61F6-27.003`, `59R-9.003` | Legacy formats vary; do not normalize — store as-is in history field |
| Subsection references | `64B16-28.109(1)` | Parse base citation then capture subsection in separate field: `\((\d+)\)` |
| Chapter-only references | `64B8-9` (no rule number) | Valid for chapter-level classification; parse as `title=64, div=B8, chapter=9, rule=null` |
| Range references | `64B8-50 through 64B8-56` | Parse as two citations and infer range |

### 6.5 Structural comparison: FAC vs. CFR

| Dimension | FAC | CFR |
|---|---|---|
| **Hierarchy** | Title → Division → Chapter → Rule | Title → Chapter → Subchapter → Part → Section |
| **Agency encoding** | Yes — agency identity is embedded (`64B8` = DOH Board of Medicine) | No — organized by subject matter, not agency |
| **Separator** | Hyphen between division and chapter | Space or "CFR" keyword between title and part |
| **Section delimiter** | Period (`.003`) | Period (`.71`) |
| **Minimum digits** | 3-digit minimum for rule number | No minimum |
| **Division concept** | Letter+number within title | Chapter within title (but "chapter" means something different) |
| **Citation prefix** | `Fla. Admin. Code R.` or `F.A.C.` | `C.F.R.` or `CFR` |
| **Classification advantage** | Agency is known from citation alone → instant A-score lookup | Agency must be inferred from title/part combination |

The FAC's embedded agency identity is a **major classification advantage**. A CitationParser receiving `64B8-9.003` can immediately determine: department = DOH, division = Medical Quality Assurance, board = Board of Medicine, topic area = Standards of Practice — all before reading any rule text. This structural signal alone enables A-score assignment (0.45) and chapter-level domain mapping (clinical-operations.practice-standards for chapter 64B8-9).

### 6.6 CitationParser interface implementation notes

```
interface FACCitation {
  raw: string;              // "64B8-9.003"
  title: number;            // 64
  division_letter: string;  // "B"
  division_number: number;  // 8
  full_division: string;    // "64B8"
  chapter: number;          // 9
  full_chapter: string;     // "64B8-9"
  rule: string;             // "003" (string to preserve leading zeros)
  full_rule: string;        // "64B8-9.003"
  subsection?: string;      // "(1)" if present
  is_emergency: boolean;    // false
  emergency_year?: number;  // null
}
```

**Lookup flow:** `parse(raw)` → extract `full_division` → lookup in agency table → get A-score → extract `full_chapter` → lookup in chapter-domain mapping → get primary domain + cross-classifications → return enriched citation with classification metadata.

### 6.7 Citation Parser Reconciliation Note

**Comparison of FACCitation to P2-S1's CitationParser Interface:**

The `FACCitation` interface defined above differs from P2-S1's `ParsedCitation` interface in several key respects:

| Aspect | P2-S1 ParsedCitation | FACCitation (this document) |
|---|---|---|
| **Jurisdiction field** | Required (`jurisdiction: string`) | Implicit (always 'FL') — recommend adding explicit `jurisdiction: 'FL'` field |
| **Title field** | `title: number` | `title: number` ✓ Compatible |
| **Part field** | `part: number` | Not applicable — FAC uses `chapter` instead |
| **Section field** | `section?: number` | `rule: string` (string to preserve leading zeros) — **type mismatch** |
| **Division concept** | Not present in federal model | `division_letter`, `division_number`, `full_division` — **FAC-specific extension** |
| **Emergency rules** | Not addressed | `is_emergency`, `emergency_year` — **FAC-specific extension** |

**Recommended alignment:** To implement FACCitation as a `CitationParser` per P2-S1's interface pattern:

```typescript
class FloridaCitationParser implements CitationParser {
  jurisdiction = 'FL';
  
  parse(raw: string): ParsedCitation | null {
    // Parse FAC format and map to ParsedCitation
    const facCitation = this.parseFAC(raw);
    if (!facCitation) return null;
    
    return {
      jurisdiction: 'FL',
      title: facCitation.title,
      // Map FAC 'chapter' to generic 'part' for interface compatibility
      part: facCitation.chapter,
      // Convert rule string to number if needed, or extend interface
      section: facCitation.rule ? parseInt(facCitation.rule) : undefined,
      // FAC-specific fields as extension properties
      _fac: {
        division_letter: facCitation.division_letter,
        division_number: facCitation.division_number,
        full_division: facCitation.full_division,
        is_emergency: facCitation.is_emergency,
        emergency_year: facCitation.emergency_year
      }
    };
  }
}
```

**Gap:** P2-S1's `ParsedCitation` interface may need extension to support state-specific fields (division hierarchy, emergency rule metadata) without breaking the generic interface contract. Consider adding an optional `extensions?: Record<string, unknown>` field to the base interface.

## Conclusion: implementation priorities and key insights

Three findings reshape how Cedar should approach Florida healthcare regulation classification. First, **the FAC citation structure is itself a classification engine** — the embedded agency hierarchy gives Cedar an A-score and chapter-level domain mapping from the citation string alone, before any text analysis. This is more powerful than CFR classification, where agency must be inferred. Second, **only four chapter families require rule-level decomposition** (64B8-9, 64B15-14, 64B16-27, 64B16-28) because they span multiple domains within a single chapter; the remaining 140+ chapters classify cleanly at the chapter level with High reliability. Third, the five structured metadata fields on every flrules.org rule page — especially "Rulemaking Authority" and "Law Implemented" — provide free, machine-readable signals that can confirm or refine chapter-level classifications without text analysis.

For implementation, the recommended priority order is: (1) load the chapter-to-domain mapping table from Section 1 (covers ~85% of classification needs); (2) implement the citation parser from Section 6 to enable structural classification; (3) add the 47 FL-specific keywords from Section 3 to handle rule-level disambiguation in Low-reliability chapters; (4) implement the 18 cross-classification patterns from Section 4; and (5) add federal reference detection from Section 5 as a confidence booster. The A-score table fromSection 2 should be loaded first as it is a prerequisite for the composite scoring model.
